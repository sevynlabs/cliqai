  <action>
    **1. Create HandoffService** at `apps/api/src/modules/crm/handoff/handoff.service.ts`:
    Injectable service with RedisService and PrismaService injected, plus AgentService (forwardRef).

    - `takeOver(organizationId: string, remoteJid: string, userId: string): Promise<void>`
      - Set Redis key `mutex:conversation:${organizationId}:${remoteJid}` to `human:${userId}` with EX 86400 (24h TTL safety net)
      - Update Conversation status to 'human_handling' via Prisma
      - Create LeadTimeline entry: { eventType: 'handoff_takeover', description: `Atendente assumiu a conversa` }
      - Return the lead with annotations for context display (HAND-02)

    - `returnToAI(organizationId: string, remoteJid: string): Promise<void>`
      - Delete Redis key `mutex:conversation:${organizationId}:${remoteJid}`
      - Call `agentService.returnToAgent(threadId)` where threadId = `${organizationId}:${remoteJid}`
      - Update Conversation status to 'active'
      - Create LeadTimeline entry: { eventType: 'handoff_return', description: `Conversa devolvida para IA` }

    - `getMutexHolder(organizationId: string, remoteJid: string): Promise<string | null>`
      - GET Redis key, return value or null

    **2. Create HandoffController** at `apps/api/src/modules/crm/handoff/handoff.controller.ts`:
    Controller with `@Controller('handoff')`:

    - POST /handoff/:conversationId/takeover — Loads conversation by ID (with org check), calls takeOver. Returns lead with annotations. Uses `@OrgRoles('owner', 'admin', 'manager', 'attendant')`.
    - POST /handoff/:conversationId/return — Calls returnToAI. Returns { success: true }. Same RBAC.
    - GET /handoff/:conversationId/status — Returns mutex holder info. Same RBAC.

    **3. Add Redis mutex check to MessageProcessor**:
    In the `process()` method, AFTER dedup check and conversation/lead upsert, BEFORE the `agentService.processMessage()` call, add:
    ```typescript
    const mutexKey = `mutex:conversation:${tenantId}:${key.remoteJid}`;
    const mutexHolder = await this.redis.get(mutexKey);
    if (mutexHolder) {
      this.logger.log(`Skipping AI for ${key.remoteJid} — mutex held by: ${mutexHolder}`);
      return; // Acknowledge job, do NOT re-throw
    }
    ```
    This is the core HAND-04 safety mechanism. Jobs already in queue when takeover happens will check mutex and exit gracefully.

    **4. Update CrmModule**: Add HandoffService to providers/exports, HandoffController to controllers.

    **5. Update AgentModule** if needed for HandoffService to access AgentService (forwardRef pattern).
