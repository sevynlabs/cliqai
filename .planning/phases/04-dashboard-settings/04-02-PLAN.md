Existing API endpoints:
- GET /api/leads — list leads with filters (stage, source, procedure, dateFrom, dateTo, scoreMin, scoreMax, tags, page, limit)
- GET /api/leads/search?q=term — global search
- GET /api/leads/:id — lead detail with annotations, timeline, appointments
- PATCH /api/leads/:id/stage — update lead stage
- GET /api/appointments — list appointments for org
- GET /api/appointments/lead/:leadId — appointments for a lead
- POST /api/appointments/:id/confirm — confirm appointment
- POST /api/appointments/:id/cancel — cancel appointment

Existing Prisma models:
- Conversation: id, organizationId, remoteJid, isHandoff, lastMessageAt, messageCount, createdAt, leadId
- Lead: id, name, phone, email, procedureInterest, stage, score, tags, bantBudget/Authority/Need/Timeline
- Appointment: id, leadId, professionalName, procedureName, startTime, endTime, status, calendarEventId
- LeadAnnotation: id, leadId, type (summary|objection|next_steps|note), content, createdAt

HandoffController endpoints (from apps/api/src/modules/crm/handoff/):
- POST /api/handoff/:conversationId/takeover — human takes over
- POST /api/handoff/:conversationId/return — return to AI
