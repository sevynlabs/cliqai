       - Map Better Auth organization roles to CliniqAI permission sets

    6. **CurrentUser decorator** (`apps/api/src/common/auth/current-user.decorator.ts`):

       - Skip for public routes: /api/health, /api/auth/*

    8. **Tenants module** (`apps/api/src/modules/tenants/`):

         - `getClinic(clinicId)`: Returns clinic details
       - `TenantsController`:
         - POST /api/tenants — create clinic (signup flow calls this after user creation)
         - GET /api/tenants/current — get current clinic details
       - Protected by AuthGuard, owner-only for create

    9. **Users module** (`apps/api/src/modules/users/`):
       - `UsersService`:
         - `inviteUser(clinicId, { email, role })`: Uses Better Auth organization invitation
         - `listMembers(clinicId)`: Lists organization members
         - `updateRole(clinicId, userId, newRole)`: Updates member role
       - `UsersController`:
         - POST /api/users/invite — invite user (owner/admin only)
         - GET /api/users — list clinic members
         - PATCH /api/users/:id/role — update role (owner only)
       - Protected by AuthGuard + @Roles decorator
