    1. **Update Prisma schema** for Better Auth compatibility:
       - Better Auth with Prisma adapter creates its own tables (user, session, account, verification, organization, member, invitation). Review Better Auth Prisma schema requirements and add them to schema.prisma.
       - Remove the custom `User` model from Plan 02 and use Better Auth's user table. Add business fields (role enum) as extensions if the adapter supports it, or create a separate `clinic_profiles` table linking Better Auth user.id to clinic-specific data.
       - Run `prisma migrate dev --name add-better-auth-tables` and append RLS to new tenant-scoped tables.

    2. **Better Auth config** (`apps/api/src/common/auth/auth.config.ts`):
       - Follow research Pattern 4 exactly
       - `betterAuth()` with `prismaAdapter(prisma, { provider: 'postgresql' })`
       - emailAndPassword enabled, requireEmailVerification: false (dev mode)
       - sendResetPassword callback: log URL to console in dev (real email in production)
       - organization plugin with:
         - `createAccessControl` defining resources: clinic, leads, conversations, settings, users
         - Roles: owner (full access), manager (leads CRUD, conversations, settings read), attendant (conversations read+respond, leads read)
         - sendInvitationEmail callback: log invite link to console in dev
       - Export `auth` instance and `Session` type

    3. **Auth module** (`apps/api/src/common/auth/auth.module.ts`):
       - Use `@thallesp/nestjs-better-auth` to register Better Auth with NestJS
       - If the community package is unavailable or incompatible, create a custom NestJS module that:
         - Mounts Better Auth handler on `/api/auth/*` routes
         - Provides AuthGuard that validates session via `auth.api.getSession()`
         - Exports @CurrentUser() decorator

    4. **RBAC guard** (`apps/api/src/common/auth/rbac.guard.ts`):
       - NestJS CanActivate guard using custom `@Roles()` decorator
       - Reads user's role from Better Auth session (organization member role)
       - Checks if user's role has required permissions for the endpoint
       - Returns 403 if insufficient permissions

    5. **Roles definition** (`apps/api/src/common/auth/roles.ts`):
       - Export role hierarchy: OWNER > ADMIN > MANAGER > ATTENDANT
       - Export `@Roles('owner')`, `@Roles('manager')` decorator for controllers
       - Map Better Auth organization roles to CliniqAI permission sets

    6. **CurrentUser decorator** (`apps/api/src/common/auth/current-user.decorator.ts`):
       - createParamDecorator that extracts user + session from request
       - Returns typed user object with id, email, name, role, activeOrganizationId

    7. **Update TenantMiddleware**:
       - Replace temporary `x-tenant-id` header with Better Auth session's `activeOrganizationId`
       - Call `auth.api.getSession({ headers: req.headers })` to resolve session
       - Store resolved tenantId in ClsService
       - Skip for public routes: /api/health, /api/auth/*

    8. **Shared auth types** (`packages/shared/src/types/auth.ts`):
        - LoginInput, SignupInput, InviteUserInput interfaces
        - CliniqRole type: 'owner' | 'admin' | 'manager' | 'attendant'
        - Permission type unions

    9. **Shared auth schemas** (`packages/shared/src/schemas/auth.schema.ts`):
        - loginSchema: email (email format), password (min 8 chars)
        - signupSchema: email, password, name (min 2 chars), clinicName (min 2 chars)
        - inviteUserSchema: email, role (CliniqRole)

