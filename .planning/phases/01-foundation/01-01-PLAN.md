  <action>
    Create the full monorepo from scratch following the research-documented structure:

    1. **Root config:**
       - `pnpm-workspace.yaml` declaring `apps/*` and `packages/*`
       - Root `package.json` with scripts: `dev` (pnpm --parallel run dev), `build` (shared first, then parallel), `typecheck`, `lint`
       - `tsconfig.base.json` with strict mode, module: "esnext", moduleResolution: "bundler", target: "ES2022"
       - `biome.json` with default rules (formatter + linter)
       - `.env.example` with placeholders: DATABASE_URL, REDIS_URL, BETTER_AUTH_SECRET, NEXT_PUBLIC_API_URL, WEB_URL, PORT, PGCRYPTO_KEY
       - `.gitignore` covering node_modules, .env, dist, .next, prisma client generated

    2. **apps/web (Next.js 16):**
       - Initialize with `package.json` (name: "@cliniq/web"), dependencies: next@^16, react@^19, react-dom@^19
       - `next.config.ts` with `transpilePackages: ['@cliniq/shared']` for workspace package support
       - `app/layout.tsx` with Plus Jakarta Sans (headings) + Inter (body) via next/font/google, teal-600 CSS variable for primary color
       - `app/page.tsx` simple landing showing "CliniqAI" heading confirming app works
       - `app/globals.css` with Tailwind v4 directives and CSS custom properties for design system (--color-primary: #0D9488)
       - `middleware.ts` as empty placeholder (auth middleware added in Plan 03)
       - Dev script: `next dev --turbopack`

    3. **apps/api (NestJS):**
       - Initialize with `package.json` (name: "@cliniq/api"), dependencies: @nestjs/core, @nestjs/common, @nestjs/platform-express, @nestjs/config, helmet
       - `src/main.ts` with `bodyParser: false` (CRITICAL for Better Auth in Plan 03), helmet, ValidationPipe with whitelist+transform, global prefix 'api', port from env (default 3001)
       - `src/app.module.ts` with ConfigModule.forRoot (isGlobal, validate env with Zod schema), HealthModule
       - `src/health/health.controller.ts` with GET /api/health returning { status: 'ok', timestamp }
       - `src/health/health.module.ts`
       - Dev script: `nest start --watch`
       - Add `@cliniq/shared: "workspace:*"` and `@cliniq/database: "workspace:*"` as dependencies

    4. **packages/shared:**
       - `package.json` (name: "@cliniq/shared"), main/types entry points
       - `src/types/index.ts` with initial UserRole enum (OWNER, ADMIN, MANAGER, ATTENDANT) and placeholder interfaces
       - `src/index.ts` barrel export
       - Build with tsc to support both ESM consumers (Next.js) and CJS consumers (NestJS): set `"module": "commonjs"` in tsconfig, output to dist/
       - Add build script and dev (watch) script

    5. **packages/database:**
       - `package.json` (name: "@cliniq/database"), dependencies: prisma, @prisma/client
       - Minimal `prisma/schema.prisma` with PostgreSQL datasource, generator client, and a placeholder Clinic model (full schema in Plan 02)
       - Export PrismaClient from index.ts
       - Scripts: `generate` (prisma generate), `push` (prisma db push), `studio` (prisma studio)

    6. **Run `pnpm install` to link all workspaces.**

    IMPORTANT anti-patterns to avoid:
    - Do NOT use ESM-only in packages/shared (NestJS needs CJS). Use tsconfig `module: "commonjs"` with `moduleResolution: "node"` for the shared package.
    - Do NOT forget `transpilePackages` in next.config.ts — without it, Next.js cannot import workspace packages.
    - Do NOT set bodyParser to true or omit it in NestJS main.ts — Better Auth REQUIRES bodyParser: false.
