  <action>
    1. **Auth client** (`apps/web/lib/auth-client.ts`):
       - `createAuthClient` from 'better-auth/client' with `baseURL: process.env.NEXT_PUBLIC_API_URL`
       - Import and add `organizationClient()` plugin
       - Export `authClient` and inferred `Session` type
       - Export convenience hooks: `useSession` (if Better Auth provides React hooks)

    2. **API client** (`apps/web/lib/api.ts`):
       - Typed fetch wrapper that calls NestJS API at NEXT_PUBLIC_API_URL
       - Includes credentials: 'include' for cookie-based auth
       - Generic `apiClient.get<T>()`, `apiClient.post<T>()` methods
       - Handles 401 by redirecting to /login

    3. **Auth layout** (`apps/web/app/(auth)/layout.tsx`):
       - Centered card layout with CliniqAI logo/heading
       - Teal-600 accent, white card on light gray background
       - No sidebar or navigation — clean auth-focused layout

    4. **Login page** (`apps/web/app/(auth)/login/page.tsx` + `components/auth/login-form.tsx`):
       - Client component form with email + password fields
       - Uses react-hook-form + Zod (loginSchema from @cliniq/shared)
       - Calls `authClient.signIn.email({ email, password })` on submit
       - Shows error toast on failure, redirects to /dashboard on success
       - Link to /signup and /forgot-password
       - Styled with Tailwind: teal primary button, clean input fields

    5. **Signup page** (`apps/web/app/(auth)/signup/page.tsx` + `components/auth/signup-form.tsx`):
       - Client component form with name + email + password + clinic name fields
       - Uses react-hook-form + Zod (signupSchema from @cliniq/shared)
       - Flow: `authClient.signUp.email()` -> then POST /api/tenants to create clinic
       - Shows error toast on failure, redirects to /dashboard on success
       - Link to /login

    6. **Forgot password page** (`apps/web/app/(auth)/forgot-password/page.tsx`):
       - Email input field
       - Calls `authClient.forgetPassword({ email })` on submit
       - Shows success message "Check your email for reset link"
       - Link back to /login

    7. **Dashboard layout** (`apps/web/app/(dashboard)/layout.tsx`):
       - Server component that checks session via auth client
       - If no session, redirect to /login
       - Basic shell: sidebar placeholder (left), main content area (right)
       - Mobile-responsive: sidebar collapses on small screens
       - Shows user name and clinic name in header

    8. **Dashboard home** (`apps/web/app/(dashboard)/page.tsx`):
       - Simple welcome page: "Welcome to {clinicName}"
       - Shows current user role
       - Placeholder cards for Phase 4 KPIs
       - This is the landing page after login

    9. **Next.js middleware** (`apps/web/middleware.ts`):
       - Check for Better Auth session cookie
       - Protected paths: /dashboard/* — redirect to /login if no session
       - Public paths: /login, /signup, /forgot-password, /api/*
       - If authenticated user visits /login, redirect to /dashboard

    Install frontend dependencies:
    ```
    pnpm --filter @cliniq/web add better-auth react-hook-form @hookform/resolvers zod
    ```

    Design system notes:
    - Primary color: teal-600 (#0D9488) for buttons, links, accents
    - Font: Plus Jakarta Sans for headings (font-heading class), Inter for body (font-body class)
    - Clean, clinical aesthetic — no playful elements
    - All forms show inline validation errors below fields
