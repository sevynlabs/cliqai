1. **sdr.state.ts**: Define `SDRState` using `Annotation.Root()` from `@langchain/langgraph`:
   - `messages`: BaseMessage[] with messagesStateReducer (appends new messages)
   - `organizationId`: string
   - `remoteJid`: string
   - `instanceName`: string
   - `leadName`: string | null (default null)
   - `procedureInterest`: string | null (default null)
   - `consentGiven`: boolean (default false)
   - `qualificationStage`: 'consent' | 'greet' | 'qualify' | 'objection' | 'schedule' | 'handoff' (default 'consent')
   - `bantScore`: { budget: boolean, authority: boolean, need: boolean, timeline: boolean } (all default false)
   - `turnCount`: number (default 0)
   - `consecutiveUnresolved`: number (default 0)
   - `humanHandoffRequested`: boolean (default false)
   - `agentConfig`: object | null (default null) — shape matches AgentConfig Prisma model fields
   All non-messages fields use `value: (_, b) => b` reducer (latest value wins).
   Export the SDRState type and the state annotation.

2. **redis-checkpointer.ts**: Factory function `createRedisCheckpointer(redisUrl: string): Promise<RedisSaver>`. Uses `RedisSaver.fromUrl(redisUrl)` from `@langchain/langgraph-checkpoint-redis`. Export the factory.

3. **system.prompt.ts**: Export functions that build system prompts from AgentConfig:
   - `buildConsentPrompt()`: Returns the LGPD consent message in PT-BR. Must include: clinic name placeholder, what data will be collected, purpose (agendamento), right to refuse, how to consent ("responda SIM"). Must NOT include any BANT questions.
   - `buildGreetingPrompt(config: AgentConfig)`: Returns system prompt for greeting node. Uses config.personaName, config.tone, config.specialtyText. Instructs agent to introduce itself, ask how it can help, be warm but brief.
   - `buildQualifyPrompt(config: AgentConfig, bantScore)`: Returns system prompt for qualification. Instructs agent to naturally extract BANT data through conversation (not interrogation). Includes which BANT fields are still missing. Uses config.tone. Instructs: NEVER diagnose, NEVER promise results, NEVER mention prices unless asked. Last 8 messages only as conversation context (not full history).

4. **consent-check.node.ts**: Node function `consentCheckNode(state, config)`:
   - If `state.consentGiven === true`: return empty update (pass through).
   - Otherwise: Check if the user's latest message contains affirmative consent ("sim", "aceito", "concordo", "ok", "pode", "autorizo" — case insensitive).
   - If affirmative: Call lgpdService.recordConsent() with organizationId, remoteJid, channel 'whatsapp'. Return `{ consentGiven: true, qualificationStage: 'greet' }`.
   - If NOT affirmative AND this is the first message (messages.length <= 1): This is the very first contact. Return a partial update that keeps qualificationStage at 'consent'. The agent.service will send the LGPD consent message (from buildConsentPrompt).
   - If NOT affirmative AND messages.length > 1: User declined or sent something else. Return `{ qualificationStage: 'consent' }` — agent re-asks politely.
   
   The node receives lgpdService via closure (factory pattern — the node function is created inside AgentModule.onModuleInit with injected dependencies).

5. **greet.node.ts**: Node function `greetNode(state, config)`:
   - Build system prompt via `buildGreetingPrompt(state.agentConfig)`.
   - Call ChatAnthropic (claude-3-5-sonnet model) with system prompt + last 4 messages from state.
   - Return `{ messages: [aiMessage], qualificationStage: 'qualify', turnCount: state.turnCount + 1 }`.
   - ChatAnthropic uses ANTHROPIC_API_KEY env var. Model: 'claude-sonnet-4-20250514'. Temperature: 0.7. Max tokens: 300.

6. **qualify.node.ts**: Node function `qualifyNode(state, config)`:
   - Build system prompt via `buildQualifyPrompt(state.agentConfig, state.bantScore)`. Include instruction to output a JSON block at the end of its response with any BANT updates: `{"bant_update": {"budget": true/false, ...}, "lead_name": "...", "procedure_interest": "..."}`.
   - Call ChatAnthropic with system prompt + last 8 messages.
   - Parse the AI response: extract BANT update JSON if present (regex match `\{.*bant_update.*\}` from response). Strip the JSON block from the visible message.
   - Return `{ messages: [cleanedAiMessage], bantScore: mergedBant, leadName, procedureInterest, turnCount: state.turnCount + 1, consecutiveUnresolved: didExtractNewInfo ? 0 : state.consecutiveUnresolved + 1 }`.
   - If all 4 BANT fields are true, set qualificationStage to 'schedule' (Phase 3 will handle).

7. **sdr.graph.ts**: `buildSDRGraph(checkpointer, nodeFactory)` function:
   - Create StateGraph with SDRState annotation.
   - Add nodes: consent_check, greet, qualify. (Plan 02-03 adds objection, ethics_guard, loop_guard, human_handoff.)
   - Edges:
     - START -> consent_check
     - consent_check -> conditional: if consentGiven -> greet; else -> END (waiting for consent reply)
     - greet -> qualify
     - qualify -> conditional: if all BANT true -> END (qualified — Phase 3 handles scheduling); if humanHandoffRequested -> END; else -> END (wait for next user message)
   - Compile with checkpointer. Return compiled graph.
   
   Note: Each node invocation ends with END because WhatsApp is async — the user sends a message, we process ONE turn, send ONE reply, and wait. The next user message triggers a new invoke which restores state from Redis. This is NOT a chatbot loop — it's request-response per message.

IMPORTANT: Nodes need access to NestJS services (LgpdService, WhatsappService). Use a factory pattern: AgentService.onModuleInit() creates node functions as closures that capture injected services, then builds the graph. Do NOT try to use NestJS DI inside LangGraph nodes directly.

1. **agent.service.ts**: Injectable service with OnModuleInit lifecycle hook.
   - Inject: PrismaService, LgpdService, WhatsappService, ConfigService, RedisService.
   - `onModuleInit()`:
     - Create Redis checkpointer via `createRedisCheckpointer(configService.get('REDIS_URL'))`.
     - Create node functions as closures capturing injected services (lgpdService for consent_check, etc.).
     - Build and compile graph via `buildSDRGraph(checkpointer, nodes)`. Store as `this.compiledGraph`.
   - `processMessage(tenantId: string, instanceName: string, payload: EvolutionWebhookDto)`:
     - Extract text from payload: `payload.data.message.conversation || payload.data.message.extendedTextMessage?.text`.
     - Build threadId: `${tenantId}:${payload.data.key.remoteJid}`.
     - Load agentConfig from DB: `prisma.forTenant(tenantId).agentConfig.findUnique({ where: { organizationId: tenantId } })`. If null, use defaults (personaName: 'Sofia', tone: 'informal', etc.).
     - Invoke graph: `this.compiledGraph.invoke({ messages: [new HumanMessage(text)], organizationId: tenantId, remoteJid: payload.data.key.remoteJid, instanceName, agentConfig }, { configurable: { thread_id: threadId } })`.
     - Extract last AI message from result.messages (find last message where `_getType() === 'ai'`).
     - If reply exists AND result.humanHandoffRequested is false AND result.qualificationStage !== 'consent' (consent message sent by node itself):
       - Call `whatsappService.sendText(instanceName, payload.data.key.remoteJid, reply.content)`.
     - If result.qualificationStage === 'consent' AND !result.consentGiven:
       - Send LGPD consent message via `whatsappService.sendText(instanceName, remoteJid, buildConsentPrompt())`. This is the first-contact scenario.
     - Update Conversation record in DB: turnCount, qualificationStage, status.
   - `returnToAgent(threadId: string)`: For future human handoff return. Calls `this.compiledGraph.updateState({ configurable: { thread_id: threadId } }, { humanHandoffRequested: false, qualificationStage: 'qualify' })`.
