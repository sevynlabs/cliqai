1. **evolution-api.client.ts**: Injectable class using axios with base URL from `EVOLUTION_API_URL` env var and `apikey` header from `EVOLUTION_API_KEY`. Methods:
   - `createInstance(instanceName: string, webhookUrl: string)`: POST /instance/create with { instanceName, webhookUrl, webhookByEvents: false, events: ['MESSAGES_UPSERT', 'CONNECTION_UPDATE', 'QRCODE_UPDATED'], qrcode: true }. Returns { instanceName, qrcode: string }.
   - `getConnectionState(instanceName: string)`: GET /instance/connectionState/{instanceName}. Returns 'open' | 'close' | 'connecting'.
   - `deleteInstance(instanceName: string)`: DELETE /instance/delete/{instanceName}.
   - `sendText(instanceName: string, to: string, text: string)`: POST /message/sendText/{instanceName} with { number: to, text }.
   - `sendMedia(instanceName: string, to: string, mediaType: 'image'|'document'|'audio', url: string, caption?: string)`: POST /message/sendMedia/{instanceName}.
   Add axios retry interceptor (3 retries, exponential backoff 1s/2s/4s) for 5xx errors only.

2. **evolution-webhook.dto.ts**: Class with class-validator decorators matching Evolution API v2 payload: `event` (string — 'messages.upsert' | 'connection.update' | 'qrcode.updated'), `instance` (string — instance name, NOT id), `data` (object with nested `key: { remoteJid: string, fromMe: boolean, id: string }`, `message?: { conversation?: string, extendedTextMessage?: { text: string } }`, optional `state?: string` for connection events, optional `qrcode?: { base64: string }` for QR events). Use @ValidateNested and @Type decorators for nested objects.

3. **send-message.dto.ts**: DTO for outbound message job data: instanceName, to (remoteJid), text, mediaType?, mediaUrl?, caption?.

4. **whatsapp.controller.ts**: 
   - `POST /api/whatsapp/webhook` — @Public() (no auth — validated by instance lookup). Receives EvolutionWebhookDto. Routes by event type:
     - `messages.upsert`: Filter out `fromMe === true` immediately. Filter non-text messages (no `conversation` and no `extendedTextMessage`). Resolve tenantId via `whatsappService.resolveTenantByInstance(payload.instance)`. If unknown instance, return `{ status: 'unknown_instance' }`. Enqueue to `whatsapp.inbound` with `jobId: payload.data.key.id` for deduplication. Return `{ status: 'queued' }`.
     - `connection.update`: Call `whatsappService.handleConnectionUpdate(payload.instance, payload.data.state)`.
     - `qrcode.updated`: Call `whatsappService.handleQrCodeUpdate(payload.instance, payload.data.qrcode.base64)`.
   - `POST /api/whatsapp/instances` — @MinRole('owner'). Creates Evolution API instance for current tenant. Instance name = `{organizationId}-wa`. Saves WhatsappInstance record. Returns QR code.
   - `GET /api/whatsapp/instances/status` — @MinRole('attendant'). Returns current connection status from DB.
   - `DELETE /api/whatsapp/instances` — @MinRole('owner'). Deletes instance from Evolution API and DB.

5. **whatsapp.service.ts**: Injectable service. Injects PrismaService, EvolutionApiClient, and outbound Queue.
   - `resolveTenantByInstance(instanceName: string)`: Query WhatsappInstance by instanceName, return organizationId or null.
   - `handleConnectionUpdate(instanceName, state)`: Update WhatsappInstance.status in DB. If state === 'close', enqueue reconnection job with exponential backoff (attempt 1: 5s, 2: 15s, 3: 45s, 4: 135s, 5: 405s). After 5 failures, enqueue notification to 'notifications' queue.
   - `handleQrCodeUpdate(instanceName, base64)`: Update WhatsappInstance.qrCodeBase64 in DB.
   - `sendText(instanceName, to, text)`: Enqueue to `whatsapp.outbound` queue with random delay 100-800ms (jitter). Do NOT call Evolution API directly — goes through rate-limited outbound worker.
   - `sendMedia(instanceName, to, mediaType, url, caption?)`: Same pattern via outbound queue.
   - `createInstance(organizationId)`: Call EvolutionApiClient.createInstance, save WhatsappInstance to DB, return QR code.
   - `getStatus(organizationId)`: Return WhatsappInstance for tenant.
   - `deleteInstance(organizationId)`: Delete from Evolution API and DB.

6. **message.processor.ts**: @Processor('whatsapp.inbound'). Injects PrismaService. `process(job)` method:
   - Extract tenantId, instanceName, payload from job.data.
   - Secondary deduplication: Redis SET NX `dedup:{payload.data.key.id}` with 24h TTL. If key exists, skip (already processed).
   - Upsert Conversation record (by organizationId + remoteJid): create if not exists, increment turnCount.
   - Call `agentService.processMessage(tenantId, instanceName, payload)` — for THIS plan, stub AgentService as a placeholder that logs "Agent processing not yet implemented" and does NOT send any WhatsApp reply. Plan 02-02 will implement the real AgentService.

7. **whatsapp.outbound processor**: Either in message.processor.ts or separate file. @Processor('whatsapp.outbound'). Worker config: `limiter: { max: 1, duration: 1500 }`. Calls EvolutionApiClient.sendText or sendMedia based on job data.

8. **whatsapp.module.ts**: NestJS module importing QueueModule, PrismaModule, RedisModule. Provides EvolutionApiClient, WhatsappService. Registers processors.

9. **app.module.ts**: Add WhatsappModule to imports.

IMPORTANT: The webhook endpoint MUST be @Public() — Evolution API cannot authenticate. Validation is done by instance lookup (unknown instance = rejected).
